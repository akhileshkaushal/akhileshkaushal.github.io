HiPlex Demultiplexer

Author: Akhilesh Kaushal
Version: 1.0.0

Overview

This pipeline is designed for demultiplexing HiPlex sequencing reads by identifying custom barcode combinations embedded in the FASTQ read headers. It extracts these indices and separates reads into sample-specific FASTQ files using a user-provided index mapping table.

Features

  • Supports dual-index barcoding (i5 + i7)
  • Matches reads to samples using a barcode CSV file
  • Outputs gzip-compressed FASTQ files per sample
  • Handles mismatches and unassigned reads with optional logging
  • Docker-compatible file handling and input resolution

Input Requirements

  • Directory of raw .fastq.gz files
  • Index mapping file (CSV format with sample, i5, i7 columns)

Output

  • Sample-level demultiplexed FASTQ files (R1/R2)
  • Log file summarizing unassigned reads (optional)

Example Usage

python hiplex_demux.py --input fastq_dir --output output_dir --index-table indices.csv
